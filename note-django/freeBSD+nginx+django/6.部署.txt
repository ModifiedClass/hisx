1.安装uwsgi
pip install uwsgi

2.在django项目settings.py同目录下新建uwsgi.ini文件，内容如下：
[uwsgi]
socket = 127.0.0.1:8009
# 配置项目目录（此处设置为项目的根目录）
chdir = /www/djangopro
# 指定虚拟环境的目录
home=/root/env1
# 将项目配置文件导入项目
env1 = DJANGO_SETTINGS_MODULE=djangopro.settings
# 配置入口模块 (django的入口函数的模块，即setting同级目录下的wsgi.py)
wsgi-file = djangopro/wsgi.py
# 开启master, 将会多开一个管理进程, 管理其他服务进程
master = True
# 服务器开启的进程数量
processes = 2
# 以守护进程方式提供服, 输出信息将会打印到log中
daemonize = wsgi.log
# 服务器进程开启的线程数量
threads = 4
# 退出的时候清空环境变量
vacuum = true
# 进程pid
pidfile = uwsgi.pid

保存退出
启动uwsgi
uwsgi –ini=uwsgi.ini #ini前面是两个减号

uwsgi开机启动配置
进入/usr/local/etc/rc.d/目录
新建一个sh文件，比如我的my.sh
编辑内容为：
!/bin/sh
/usr/local/bin/uwsgi –ini /djangopro/uwsgi.ini
保存退出
编辑开机启动
ee /etc/rc.conf
增加一行
mysh=”YES”
保存退出即可。
如果执行不了需要授执行权限
chmod x my.sh

配置nginx

ee /usr/local/etc/nginx/nginx.conf

server {
listen 8000;
server_name www.***.com;
….

location / {
include uwsgi_params;　　# 设置将所有请求转发给uwsgi服务器处理
uwsgi_pass 127.0.0.1:8009;　　# 指定uwsgi服务器url端口号与uwsgi配置的socket端口号一致
uwsgi_send_timeout 600;
uwsgi_connect_timeout 600;
uwsgi_read_timeout 600;
}
location /static {
alias /www/djangopro/static/;　　# 设置将/static的静态请求交给nginx，并指定静态文件的目录
}

最后重启nginx
service nginx reload
通过地址就可以访问网站了。
